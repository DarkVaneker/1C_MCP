# 1C_MCP — MCP-сервер для 1С:Предприятие

Расширение конфигурации 1С, которое превращает информационную базу в **MCP-сервер** (Model Context Protocol) для интеграции с AI-ассистентами: Cursor, Claude Desktop и другими MCP-клиентами.

Проект основан на [vladimir-kharin/1c_mcp](https://github.com/vladimir-kharin/1c_mcp) и расширяет его двумя инструментами для **выполнения и проверки кода** на встроенном языке 1С непосредственно в контексте вашей базы.

## Отличия от базового проекта

| Возможность | [1c_mcp](https://github.com/vladimir-kharin/1c_mcp) | 1C_MCP (этот репозиторий) |
|---|---|---|
| MCP Tools / Resources / Prompts | ✓ | ✓ |
| Работа с метаданными конфигурации | ✓ | ✓ |
| **`execute_1c_code_simple`** — быстрое выполнение фрагмента кода | — | ✓ |
| **`execute_1c_code_complex`** — сборка и выполнение многофункционального модуля | — | ✓ |
| Асинхронная очередь сборки `.epf` через регламентное задание | — | ✓ |

## Как это работает

```
AI-клиент (Cursor) ←→ Python MCP Proxy ←→ HTTP-сервис mcp_APIBackend ←→ Обработки-контейнеры
```

- **HTTP-сервис** `mcp_APIBackend` реализует JSON-RPC API протокола MCP (`tools/list`, `tools/call`, `resources/*`, `prompts/*`).
- **Обработки-контейнеры** регистрируют инструменты через подсистему `mcp_КонтейнерыИнструментов`.
- Для подключения AI-клиента рекомендуется использовать **Python-прокси** из базового проекта — он решает вопросы аутентификации и транспорта `stdio`.

Подробнее об архитектуре — в [AGENTS.md](AGENTS.md).

## Инструменты MCP

### Из базового проекта

| Инструмент | Описание |
|---|---|
| `list_metadata_objects` | Список объектов метаданных по типу и маске имени |
| `get_metadata_structure` | Структура выбранного объекта метаданных |
| `get_report_details` | СКД и модуль объекта варианта отчёта |

### Новые инструменты

#### `execute_1c_code_simple`

Синхронное выполнение простого, самодостаточного фрагмента кода 1С в текущем сеансе.

**Параметры:**

| Параметр | Тип | Обязательный | Описание |
|---|---|---|---|
| `code` | string | да | Код на встроенном языке 1С |
| `context` | string | нет | JSON с переменными, объявляемыми перед выполнением (по умолчанию `{}`) |

**Особенности:**
- Поддерживает не более одной процедуры или функции.
- Если в коде есть `Возврат`, результат возвращается в ответе.
- Изменения в базе выполняются в транзакции с откатом.

**Пример вызова:**

```json
{
  "code": "Результат = 2 + 2;\nВозврат Результат;",
  "context": "{}"
}
```

#### `execute_1c_code_complex`

Асинхронная сборка кода во внешнюю обработку (`.epf`) и вызов указанной точки входа. Используйте, когда код содержит несколько процедур/функций и не может быть разбит на несколько вызовов `execute_1c_code_simple`.

**Параметры:**

| Параметр | Тип | Обязательный | Описание |
|---|---|---|---|
| `code` | string | да | Код модуля объекта с несколькими процедурами/функциями |
| `entry_point` | string | да | Имя экспортной процедуры или функции для вызова |
| `context` | string | нет | JSON с переменными контекста и параметрами метода (по умолчанию `{}`) |

**Архитектура выполнения:**

```
execute_1c_code_complex
        │
        ▼
Регистр mcp_ОчередьПроверкиКода  ──►  Регламентное задание mcp_ПроверкаКода
        │                                      │
        │                              Сборка .epf (1cv8.exe)
        │                              Выполнение entry_point
        ▼                                      ▼
   Polling результата  ◄────────────  Запись результата
```

> **Важно:** для работы `execute_1c_code_complex` на сервере 1С должно быть включено регламентное задание `mcp_ПроверкаКода`, а у пользователя/ras — права на запуск `1cv8.exe` для сборки внешних обработок.

## Быстрый старт

### Требования

- Платформа 1С:Предприятие **8.3.20** и выше (режим совместимости расширения: 8.3.20)
- Опубликованный HTTP-сервис `mcp_APIBackend` на веб-сервере
- Python MCP Proxy — см. [документацию базового проекта](https://github.com/vladimir-kharin/1c_mcp)

### Шаг 1. Установка расширения

1. Соберите расширение из каталога `Расширение/` в файл `build/MCP_Сервер.cfe` (через Конфигуратор или EDT).
2. Подключите расширение **MCP_Сервер** к вашей информационной базе.
3. Убедитесь, что пользователю назначена роль `mcp_ОсновнаяРоль`.

### Шаг 2. Настройка регламентного задания

Для инструмента `execute_1c_code_complex`:

1. Откройте **Администрирование → Регламентные и фоновые задания**.
2. Найдите задание **Mcp проверка кода** (`mcp_ПроверкаКода`).
3. Включите использование задания и настройте расписание (рекомендуется: каждые **10–30 секунд**).
4. Убедитесь, что на сервере 1С доступен `1cv8.exe` (используется для сборки `.epf`).

### Шаг 3. Публикация HTTP-сервиса

Опубликуйте HTTP-сервис `mcp_APIBackend` на веб-сервере.

> **Важно:** регистр имени базы в URL должен **точно** совпадать с именем публикации (например, `base`, а не `Base`). Несовпадение регистра приводит к редиректам, которые превращают POST-запросы в GET.

Пример URL: `http://server/base/hs/mcp/`

### Шаг 4. Подключение AI-клиента

Рекомендуется подключение через Python-прокси из [vladimir-kharin/1c_mcp](https://github.com/vladimir-kharin/1c_mcp):

```
AI-клиент ←→ Python MCP Proxy ←→ HTTP-сервис 1С
```

Примеры конфигураций MCP-клиентов — в каталоге [mcp_client_settings](https://github.com/vladimir-kharin/1c_mcp/tree/main/mcp_client_settings) базового репозитория.

**Пример для Cursor** (`~/.cursor/mcp.json`):

```json
{
  "mcpServers": {
    "1c-server-direct": {
      "url": "http://localhost:PORT/mcp",
      "headers": {}
    }
  }
}
```

Конкретный порт и способ запуска прокси — см. документацию Python-прокси в базовом проекте.

## Безопасность

Инструменты `execute_1c_code_simple` и `execute_1c_code_complex` **выполняют произвольный код** в контексте информационной базы. Это мощный, но потенциально опасный механизм.

**Рекомендации:**

- Не публикуйте HTTP-сервис без аутентификации в production.
- Используйте Python-прокси с OAuth2 / Basic Auth.
- Ограничьте доступ к MCP только доверенным пользователям и средам разработки.
- Не подключайте к продуктивным базам без явной необходимости.

## Разработка собственных инструментов

1. Создайте обработку с префиксом `mcp_ИнструментИмя`.
2. Включите её в подсистему `mcp_КонтейнерыИнструментов`.
3. Реализуйте методы `ДобавитьИнструменты()` и `ВыполнитьИнструмент()` в модуле менеджера.

Подробное руководство — в [AGENTS.md](AGENTS.md).
